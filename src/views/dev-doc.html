云红包Server API

1.文档概述

本文档针对商户使用接口模式接入魔方科技红包功能进行了详细的描述，商户端程序如需要进行开发可以此文档为依据。

2.适用范围

通过接口模式接入魔方科技红包功能的合作商户。

3.调用方式

接口采用http/https连接方式进行通讯。

4.IP限制

允许对接口的请求IP进行限制。商户在开通时可以向魔方科技提供一个或者多个固定IP用于做交易和查询。

5.超时时间设定

商户接收魔方科技接口应答的超时时间不得少于60s。

6.单位说明

接口中所有交易金额和手续费的货币种类均为人民币，并采用“分”为单位。

7.域长度计算说明

所有参数的长度均以字节为单位，每个中文汉字占三个字节，每个英文字母、字符或数字占一个字节。

8.时间说明

涉及到时间类型，如果是日期类型格式默认yyyy-MM-dd，如果是时间类型格式默认yyyy-MM-dd HH:mm:ss

9.加密说明

涉及到密码加密时，都需要用到秘钥进行加密，参考附件密码加密方法。

10.签名说明

1) 以md5作为签名算法

2) 签名算法分两步：1. 参数按照字段名的ASCII码从小到大排序；2. 将包含密钥的数据串进行MD5摘要计算，签名结果中的字母统一采用大写形式

3) 假设通信密钥为3EDA7B432E238BAD1208BADC65E39B01，则签名sign为：sign=MD5(key1:value1|key2:value2|seckey:3EDA7B432E238BAD1208BADC65E39B01)

11.请求参数

所有接口请求时需传递sign（签名字符串）和parterId（分配的商户id）

获得用户令牌

url：/api/v1/redEnvelope/findUserToken.shtml

请求类型:post

表单参数

名称 类型 是否必填 说明
partnerId String 是 商户分配的id
custUid String 是 用户 Id，最大长度 64 字节。是用户在 App 中的唯一标识码，必须保证在同一个 App 内不重复，重复的用户 Id 将被当作是同一用户。
name String 是 用户名称，最大长度 128 字节。（必传）
avatarUrl String 否 用户头像 URI，最大长度 1024 字节

返回值：

名称 类型 说明
respstat String 0000成功，其他参考返回码表
respmsg String 异常时提示信息
userToken String 用户令牌。App 获取 userToken后，根据情况可选择在 App本地保留当前用户的令牌信息，如果令牌失效，还需要重新向服务器获取userToken。userToken默认永久有效，除非您手动重新获取或者在开发者后台刷新 App Secret 。
